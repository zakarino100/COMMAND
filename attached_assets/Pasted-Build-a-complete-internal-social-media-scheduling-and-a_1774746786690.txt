Build a complete internal social media scheduling and analytics dashboard 
as a new React/Vite project. Private ops tool — no public auth, no 
multi-tenant, no billing.

---

TECH STACK
- Frontend: React + Vite
- Backend: Node/Express
- Database: Supabase (SUPABASE_URL + SUPABASE_SERVICE_KEY from env)
- Scheduler: node-cron (every 5 minutes)
- Image storage: Supabase Storage bucket "post-images"
- Meta posting: Graph API v19.0
- Notifications: Discord webhook (DISCORD_WEBHOOK_URL from env)
- AI captions: OpenAI API (OPENAI_API_KEY from env)
- No authentication needed — private Replit URL

SUPABASE SCHEMA

Table: posts
- id (uuid, primary key, default gen_random_uuid())
- brand (text)
- platforms (text[])
- caption (text)
- image_url (text, nullable)
- video_url (text, nullable)
- media_format (text) — 'image' | 'video' | 'carousel' | 'text'
- link_url (text, nullable) — original URL
- link_url_tagged (text, nullable) — UTM-tagged version
- content_type (text) — 'before_after' | 'offer' | 'testimonial' | 
  'educational' | 'behind_scenes' | 'seasonal' | 'other'
- headline_variant (text, nullable) — A/B test label
- scheduled_at (timestamptz)
- status (text) — 'scheduled'|'posting'|'posted'|'failed'
- posted_at (timestamptz, nullable)
- error_message (text, nullable)
- fb_post_id (text, nullable) — store returned post ID for metrics
- ig_post_id (text, nullable)
- gbp_post_id (text, nullable)
- created_at (timestamptz, default now())

Table: post_metrics
- id (uuid, primary key)
- post_id (uuid, references posts.id)
- platform (text)
- fetched_at (timestamptz)
- likes (int, default 0)
- comments (int, default 0)
- shares (int, default 0)
- reach (int, default 0)
- impressions (int, default 0)
- clicks (int, default 0)
- engagement_rate (float) — calculated: (likes+comments+shares)/reach
- created_at (timestamptz, default now())

Table: assets
- id (uuid, primary key)
- brand (text)
- url (text) — Supabase Storage public URL
- format (text) — 'image' | 'video'
- content_type (text)
- times_used (int, default 0)
- avg_engagement (float, nullable)
- created_at (timestamptz, default now())

Table: brand_config
- id (uuid, primary key)
- brand (text, unique)
- display_name (text)
- emoji (text)
- fb_page_id (text)
- ig_account_id (text)
- gbp_location_id (text, nullable)
- brand_voice (text) — short description for AI caption generation
- created_at (timestamptz, default now())

SCHEDULER (node-cron)

Job 1 — Post scheduler (every 5 minutes):
Query posts where scheduled_at <= now() AND status = 'scheduled'
For each: set status 'posting' → fire platforms in parallel →
update status 'posted' or 'failed' → send Discord notification

Job 2 — Metrics sync (every 6 hours):
Query posts where status = 'posted' AND posted_at >= now() - 30 days
For each post, for each platform:
- Call Meta Graph API for likes, comments, shares, reach, impressions
- Call GBP API for clicks/views
- Upsert into post_metrics table
- Recalculate engagement_rate
- Update assets.avg_engagement for any asset used in that post

META POSTING

Tokens from env:
- META_TOKEN_WOLFPACKWASH
- META_TOKEN_MOPMAFIA
- META_TOKEN_BLUEOCEAN

Facebook:
- Text post: POST /{page-id}/feed {message}
- Image post: POST /{page-id}/photos {url, caption}
- Store returned post ID as fb_post_id

Instagram (two-step container):
- Step 1: POST /{ig-id}/media {image_url, caption}
- Step 2: POST /{ig-id}/media_publish {creation_id}
- Store returned ID as ig_post_id
- Skip if no media (IG feed requires image/video)

Metrics fetch (Graph API):
- GET /{post-id}/insights?metric=impressions,reach,post_clicks
- GET /{post-id}?fields=likes.summary(true),comments.summary(true),shares

---

GBP POSTING

OAuth2: GBP_CLIENT_ID, GBP_CLIENT_SECRET, GBP_REFRESH_TOKEN
Auto-refresh access token before each call.
POST to /v4/accounts/{account-id}/locations/{location-id}/localPosts
Type: STANDARD, topicType: STANDARD
Store returned name as gbp_post_id

"You write punchy, human, conversion-focused social media captions for
